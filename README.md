- How to run the project locally

Every project should be deployed somewhere. Be sure to include the link to the deployed project so that the viewer can click around and see what it's all about.

## Getting Started with the Project

### Dependency Installation & Startup Development Server

Once cloned, navigate to the project's root directory and this project uses npm (Node Package Manager) to manage its dependencies.

The command below is a combination of installing dependencies, opening up the project on VS Code and it will run a development server on your terminal.

```bash
npm i && code . && npm run dev
```

## Instructions

This is your project. You choose what to build and how and what the scope should be. Try to make a plan and think about how much time you can spend on your project.

Goals
Technical goals

✓ Frontend in React

✓ Backend in Node.js

✓ MongoDB database

✓ Navigation using React Router

✓ Global state using Context API or Zustand

✓ Should work in Chrome, Firefox & Safari

✓ Be responsive and work well on mobile, tablet and desktop (320px - 1600px)

✓ Follow our guidelines on how to write clean code

✓ Both the frontend and backend (and the database) should be deployed somewhere, and links to the frontend and backend should be added to the Readme file of the repository

## Visual goals

✓ Your application should have a clear structure, using the box model as a reference, with consistent margins/paddings.

✓ You should use consistent h1-h6 styles and sizes throughout your site and for multiple devices.

✓ You should use a colour scheme based on one or a few colours.

✓ You should have a design that can be adjusted to fit devices of all sizes. Remember that 58.43% (Mar 2023) of all users will visit your page from a mobile device and that the page should therefore be optimised for these users too!

With the technical goals in mind, we also would like to emphasise that this is your project. If you would like to make a very backend-heavy application with a small and limited frontend – go ahead. If you really would like to dig deep into React and make an advanced frontend, keep the backend to a minimum.

Working as a Web Developer you will not only have to deliver clean code and fancy functionality, but you also have to be able to deliver a product that looks good visually on multiple devices. In your final project, you should therefore add some extra thoughts to the basic design elements to make it clear it’s the flagship of your portfolio.

You don’t have to be a designer to set up a solid, nice-looking page. Consistency is key, and by setting a basic visual hierarchy, using typography correctly, and having properly distributed whitespace and a colour scheme, you get a lot for free.
